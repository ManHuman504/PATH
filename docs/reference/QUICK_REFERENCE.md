## ❓ Что реально возвращает Extension.renderUI()?

### Ответ:
**ПОЛНЫЙ HTML ДОКУМЕНТ (string), НЕ JSON!**

```typescript
async renderUI(props): Promise<string> {
  return `<!DOCTYPE html>
<html>
  <head><style>CSS...</style></head>
  <body>HTML...</body>
  <script>JavaScript...</script>
</html>`;
}
```

❌ **ДОКУМЕНТАЦИЯ НЕВЕРНА** - говорит про JSON-описание, а код возвращает HTML

📄 Источник: niceUIExtension.ts (559 строк), uglyUIExtension.ts (675 строк)

## ❓ Как РЕАЛЬНО работают модули? Как вкладки?

### Ответ:
**НЕТ! Модули НЕ работают как вкладки.**

```
❌ ДОКУМЕНТАЦИЯ ГОВОРИТ:
"ТОЛЬКО ОДИН модуль активен одновременно"
"Переход на другой модуль деактивирует предыдущий"

✅ РЕАЛЬНОСТЬ:
"ВСЕ модули активны ОДНОВРЕМЕННО"
"Нет методов: enable(), disable(), activate(), deactivate()"
"UITab - только визуальное представление, не логический контроль"
```

**Пример**:
- PathModule регистрирует обработчики CREATE_PATH, DELETE_PATH, ADD_NODE
- Эти обработчики **ВСЕГДА** активны
- Даже если пользователь переходит на другую вкладку, PathModule **продолжает обрабатывать команды**
- UITab - только способ показать пользователю какие команды доступны

📄 Источник: engine.ts (registerModule), IModule интерфейс
